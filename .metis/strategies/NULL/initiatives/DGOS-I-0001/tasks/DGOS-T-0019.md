# Reconcile daimyo onto the Artifact Protocol

## Parent Initiative

[[DGOS-I-0001]] — makes the just-built `daimyo` Loop substrate consume the protocol's generated TypeScript binding instead of its own hand-rolled artifact types, so the protocol is genuinely the single source of truth and not just an aspiration alongside a fork.

## Objective

Change `daimyo` so that the artifact types it currently hand-rolls in `daimyo/src/core/domain.ts` (`DecisionRequest`, `DecisionRecord`, `DecisionVerdict`, execution evidence, ownership-surface, touch-report — and `ValidationReport`/`RoleResult`-shaped types where present) are **imported from / generated by the `protocol` package** rather than independently defined. This is the payoff of the JSON-Schema-source-of-truth decision: one authoritative contract, with daimyo as a conforming consumer. All of daimyo's existing tests must stay green; daimyo gets a version bump.

## Acceptance Criteria

- [ ] `daimyo` takes a dependency on the `protocol` package and imports the protocol's generated TS types for the artifacts it shares with the v1 catalog, replacing the corresponding hand-rolled definitions in `daimyo/src/core/domain.ts`.
- [ ] Where daimyo's runtime needs daimyo-specific fields not in the protocol (e.g. internal execution-store bookkeeping), those are kept as daimyo-local extensions *of* the protocol types (composition), not as forks that redefine the shared shape.
- [ ] The reconciliation deltas recorded by [[DGOS-T-0015]]/[[DGOS-T-0016]]/[[DGOS-T-0017]]/[[DGOS-T-0018]] are all resolved: every intentional rename/restructure is applied to daimyo, and the result type-checks.
- [ ] **All existing daimyo tests still pass** (the full suite that was green at daimyo 0.11.0 — execution store, work-source conformance, validation, decision provider, supervisor, waves, reconciliation, standalone composition, cross-port boundary). No test is weakened to accommodate the change; if a test asserted a now-renamed field, it's updated to the protocol name.
- [ ] `daimyo` `npm run typecheck`/`lint`/`test`/`build` all clean; `src/core` remains import-pure of *adapters/siblings* — note the `protocol` dependency is a shared contract library (a type/schema dependency), and whether it counts as allowed in core is decided and documented here (recommended: allowed — it's the contract, not an adapter).
- [ ] Both plugins' versions bumped per repo rules: `daimyo` (minor) for the dependency adoption; `protocol` if its public surface changed during reconciliation. Both `package.json` + `.claude-plugin/plugin.json` (where applicable) updated; daimyo `dist/` rebuilt + committed.
- [ ] The cross-port-boundary test in daimyo is updated if needed to permit `protocol` imports in core while still rejecting `katana`/`guardrails`/`audit`.

## Implementation Notes

### Technical Approach

- This task runs only after T-0015–T-0018 have authored the schemas + bindings and recorded their daimyo-reconciliation deltas. Work artifact-by-artifact: swap daimyo's local type for the protocol type, fix call sites, run the suite.
- Decide the core-purity rule for `protocol`: the existing `cross-port-boundary.test.ts` forbids `katana`/`guardrails`/`audit` imports in `src/core`. `protocol` is the shared contract (like importing shared types), not a sibling adapter — so it should be explicitly *allowed*. Update the boundary test's allowlist and document the rationale.
- Prefer composition for daimyo-only fields: `type DaimyoDecisionRecord = ProtocolDecisionRecord & { ...daimyo-internal... }` rather than redefining.
- Keep the change mechanical and test-guarded; the suite is the safety net. If a protocol shape turns out to be wrong (not just renamed) during reconciliation, fix the *schema* (and regenerate), don't fork in daimyo.

### Dependencies

- **Upstream:** [[DGOS-T-0015]], [[DGOS-T-0016]], [[DGOS-T-0017]], [[DGOS-T-0018]] (all schemas + bindings + recorded deltas) and [[DGOS-T-0013]]/[[DGOS-T-0014]] (package + envelope).
- **Downstream:** [[DGOS-T-0020]] (compat corpus can include daimyo-produced fixtures once daimyo emits protocol-conformant artifacts).

### Risk Considerations

- **Regressions in daimyo's central runtime** (DecisionProvider/supervisor) from a type swap. Mitigation: the full daimyo suite is the gate; go artifact-by-artifact; never weaken a test to pass.
- **Schema turned out wrong, not just renamed:** Mitigation: fix the schema + regenerate (it's the source of truth) rather than forking daimyo — that's the whole point of the reconciliation.
- **Core-purity ambiguity** for the `protocol` dependency. Mitigation: decide + document (recommended: allowed as the shared contract) and encode it in the boundary test.

### Execution Profile

**Recommended Agent: opus + high.** This rewires the most central runtime package (daimyo core) onto a new shared contract while holding a large existing test suite green; subtle type mismatches ripple through the supervisor and decision provider. Cross-cutting, shipped-code-touching, and the linchpin that makes the source-of-truth decision real.
