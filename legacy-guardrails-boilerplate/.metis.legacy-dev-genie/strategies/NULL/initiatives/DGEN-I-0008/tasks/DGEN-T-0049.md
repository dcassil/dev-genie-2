# Measure edit-hook latency on reference repos

## Parent Initiative

[[DGEN-I-0008]]

## Objective

Measure end-to-end latency of `lint-edited-file.sh` on representative repos to confirm the <300ms target from DGEN-I-0008 is achievable. Results gate whether to default Q3 to "yes".

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] Measure p50/p95 hook latency on at least 2 repos (this repo + one larger reference)
- [ ] Numbers recorded in initiative or follow-up ADR
- [ ] If >1s on any reference repo, document mitigation (config tuning, ignore patterns) before proceeding to DGEN-T-0051

## Implementation Notes

Time the full hook path, not just `eslint`. Use `time` or hook-emitted timing if available. Run a representative agent edit (typical `.tsx` file) 10x and report distribution.
